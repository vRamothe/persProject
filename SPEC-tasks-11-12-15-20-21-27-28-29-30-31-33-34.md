## Task 11 — Rate Limiting on Login & Quiz

**Problem:** `ConnexionView.post` and `soumettre_quiz` / `soumettre_revisions` have no throttling. A bot can brute-force passwords or spam quiz submissions indefinitely.

**Approach:** Use `django-axes` (battle-tested, integrates with Django auth) rather than a homemade decorator. It tracks failed login attempts per IP/username, locks out after a configurable threshold, and integrates with the admin panel.

For quiz submissions (not auth), a lightweight rate-limit decorator using Django's cache backend is sufficient (no extra dependency).

### Changes

**`requirements.txt`**
```
django-axes==7.0.0
```

**`config/settings/base.py`**
```python
INSTALLED_APPS = [
    ...
    "axes",
]

AUTHENTICATION_BACKENDS = [
    "axes.backends.AxesStandaloneBackend",
    "django.contrib.auth.backends.ModelBackend",
]

MIDDLEWARE = [
    ...
    "axes.middleware.AxesMiddleware",  # must be after SessionMiddleware
]

# django-axes config
AXES_FAILURE_LIMIT = 5           # lock after 5 failed attempts
AXES_COOLOFF_TIME = 1            # 1 hour lockout
AXES_LOCKOUT_CALLABLE = "axes.helpers.get_lockout_response"
AXES_RESET_ON_SUCCESS = True
AXES_BEHIND_REVERSE_PROXY = True  # trust X-Forwarded-For (Heroku/nginx)
```

**`users/views.py`** — `ConnexionView.post` already uses Django's `login()`, which axes hooks into automatically. No change needed there beyond the settings above.

**`progress/views.py`** — add a simple cache-based rate limiter for quiz submissions:
```python
from django.core.cache import cache
from django.http import HttpResponseTooManyRequests

def _check_quiz_rate_limit(user_id):
    """Returns True if the user has exceeded 30 quiz submissions per minute."""
    key = f"quiz_rate_{user_id}"
    count = cache.get(key, 0)
    if count >= 30:
        return True
    cache.set(key, count + 1, timeout=60)
    return False
```
Called at the top of `soumettre_quiz`, `soumettre_quiz_chapitre`, and `soumettre_revisions`. Return 429 or redirect with an error message if limit exceeded.

**Migration:** `python manage.py migrate` (axes creates its own tables via migration).

### New URL
None. `axes` adds `/admin/axes/` entries automatically.

- Correct 6th attempt while locked out is still blocked even with valid credentials
- Successful login resets the axes failure counter (`AXES_RESET_ON_SUCCESS = True`)
- Lockout is per-username, not per-IP: different IP same username is still locked

## Task 12 — Email Verification on Signup

**Problem:** Anyone can register with a fake/typo email. This allows dummy accounts and makes password reset useless for those users.

**Approach:** Django's built-in `email_verification` in `PasswordResetView` re-uses the same token mechanism. We'll implement a lean custom flow:
1. On registration, set `is_active=False` on the new user.
2. Send a verification email with a signed token (`django.core.signing`).
3. A verification URL activates the account.
4. If unverified, show a "check your email" page instead of logging in.

**No new model needed** — `CustomUser.is_active` (already present in Django's `AbstractBaseUser`) is sufficient.

### Changes

**`users/models.py`** — `CustomUser` already inherits from `AbstractBaseUser` which has `is_active`. No change needed.

**`users/views.py`** — `InscriptionView.post`:

user.save()
_debloquer_premiers_chapitres(user)

_envoyer_email_verification(request, user)

New helper:
```python
from django.core import signing
from django.core.mail import send_mail
from django.urls import reverse

def _envoyer_email_verification(request, user):
    token = signing.dumps(user.pk, salt="email-verification")
    url = request.build_absolute_uri(
        reverse("verifier_email", kwargs={"token": token})
    )
    send_mail(
        subject="ScienceLycée — Confirmez votre adresse email",
        message=f"Bonjour {user.prenom},\n\nCliquez sur ce lien pour activer votre compte :\n{url}\n\nCe lien expire dans 24h.",
        from_email=None,  # uses DEFAULT_FROM_EMAIL
        recipient_list=[user.email],
    )
```

def verifier_email_view(request, token):

        user_pk = signing.loads(token, salt="email-verification", max_age=86400)

    user = get_object_or_404(CustomUser, pk=user_pk)
    user.is_active = True
    user.save()
    login(request, user, backend="django.contrib.auth.backends.ModelBackend")

**`courses/admin.py`** — add `difficulte` to `list_display` and `list_filter` on `QuestionAdmin`.

**`courses/views.py`** — `quiz_chapitre_view` currently uses `random.sample`. Replace with balanced sampling:
```python
from collections import defaultdict

def _selectionner_questions_chapitre(chapitre, nb_total=10):
    """Sélectionne questions avec mix équilibré facile/moyen/difficile."""
    toutes = list(Question.objects.filter(
        quiz__lecon__chapitre=chapitre
    ).select_related("quiz"))

    par_difficulte = defaultdict(list)
    for q in toutes:
        par_difficulte[q.difficulte].append(q)

    # Quota: 40% facile, 40% moyen, 20% difficile (arrondi)
    quotas = {"facile": 4, "moyen": 4, "difficile": 2}
    selection = []
    for diff, quota in quotas.items():
        pool = par_difficulte[diff]
        random.shuffle(pool)
        selection.extend(pool[:quota])

    # Compléter si pas assez dans une catégorie
    if len(selection) < nb_total:
        already = {q.id for q in selection}
        reste = [q for q in toutes if q.id not in already]
        random.shuffle(reste)
        selection.extend(reste[:nb_total - len(selection)])

    random.shuffle(selection)
    return selection[:nb_total]
```

**`templates/courses/quiz_chapitre.html`** — optionally show difficulty badge (🟢 Facile / 🟡 Moyen / 🔴 Difficile) next to each question.

**`templates/courses/quiz.html`** — same optional badge for lesson quizzes.

**`courses/management/commands/seed_content.py`** — update seed to assign `difficulte` values to seeded questions (distribute roughly 40/40/20).

### Tests to add (`courses/tests.py`)

**Sampling algorithm — sanity:**
- Full pool (≥4 facile, ≥4 moyen, ≥2 difficile): selection has exactly 4 facile, 4 moyen, 2 difficile
- Selection always returns exactly `nb_total` (10) questions when pool is large enough
- No duplicate question IDs in the returned list
- Selection is randomised: calling twice on the same pool returns different orders (run 20 iterations, assert not all identical)

**Sampling algorithm — fallback edge cases:**
- Zero difficile questions: 10 questions returned using facile/moyen to fill the gap
- Only 3 facile questions available: takes all 3, fills remainder from moyen/difficile
- Pool smaller than `nb_total` (6 questions total): returns all 6, no crash, no duplicates
- Empty pool: returns empty list without raising an exception

**Default value:**
- Creating a `Question` without specifying `difficulte` defaults to `"moyen"`
- Existing questions (created before this migration) default to `"moyen"`

## Task 27 — DRY Quiz Submission Logic (Opus)

**Problem:** `soumettre_quiz` (lines 44–152) and `soumettre_quiz_chapitre` (lines 195–290) share ~80% identical code: parsing `question_ids`, iterating questions, building `corrections[]`, computing score. Any bug fix or new feature (difficulty badges, instant feedback) must be applied in two places.

**Approach:** Extract a private helper `_evaluer_reponses(questions, post_data)` that returns the `corrections` list and `(points_obtenus, total_points)`. Both views call this helper.

### Changes

**`progress/views.py`** — new helper (replaces duplicated loops):
```python
def _evaluer_reponses(questions, post_data):
    """
    Évalue les réponses POST pour une liste de questions.
    Retourne (corrections, points_obtenus, total_points).
    """
    total_points = sum(q.points for q in questions)
    points_obtenus = 0
    corrections = []

    for question in questions:
        cle = f"question_{question.id}"
        reponse = post_data.get(cle, "").strip()

        correct = reponse.lower() == str(question.reponse_correcte).strip().lower()
        if not correct and question.type == "texte_libre":
            correct = _comparer_texte_libre(reponse, question)
        if correct:
            points_obtenus += question.points

        # Libellés lisibles
        if question.type == "qcm" and question.options:
            try:
                reponse_donnee_texte = question.options[int(reponse)] if reponse.isdigit() else reponse
            except (IndexError, ValueError):
                reponse_donnee_texte = reponse
            try:
                reponse_correcte_texte = question.options[int(question.reponse_correcte)]
            except (IndexError, ValueError):
                reponse_correcte_texte = question.reponse_correcte
        elif question.type == "texte_libre":
            reponse_donnee_texte = reponse if reponse else "—"
            reponse_correcte_texte = question.reponse_correcte
        else:
            reponse_donnee_texte = reponse.capitalize() if reponse else "—"
            reponse_correcte_texte = str(question.reponse_correcte).capitalize()

        corrections.append({
            "question": question,
            "reponse_donnee": reponse,
            "reponse_donnee_texte": reponse_donnee_texte,
            "reponse_correcte_texte": reponse_correcte_texte,
            "correct": correct,
        })

    return corrections, points_obtenus, total_points
```

Both `soumettre_quiz` and `soumettre_quiz_chapitre` are then simplified to call `_evaluer_reponses(questions, request.POST)` and use the returned tuple.

**No model changes, no migrations, no URL changes.**

### Tests to add (`progress/tests.py`)

**Unit tests for `_evaluer_reponses` (pure function — no DB, no HTTP):**
- QCM correct answer: `corrections[0]["correct"] == True`, `points_obtenus == question.points`
- QCM wrong answer: `correct == False`, `points_obtenus == 0`
- `vrai_faux` correct: `"vrai"` matches `"vrai"` (case-insensitive)
- `vrai_faux` wrong: `"faux"` does not match `"vrai"`
- `texte_libre` exact match: correct

- `texte_libre` case-insensitive: `"AZOTE"` matches `"azote"`
- `texte_libre` wrong with no tolerance: incorrect
- Empty response string (`""`) is always incorrect
- Multiple questions: `total_points` equals sum of all `question.points`
- `corrections` list length equals `len(questions)`
- Each `corrections` entry has keys: `question`, `reponse_donnee`, `reponse_donnee_texte`, `reponse_correcte_texte`, `correct`
- `reponses_soumises` dict built in the view contains one entry per question (keyed by `str(question.id)`)

## Task 30 — Full-Text Search

**Problem:** No search. Students can't find a specific lesson or formula without manually browsing every chapter.

**Approach:** PostgreSQL `SearchVector` + `SearchRank` via `django.contrib.postgres`. A search bar (HTMX `hx-get`) in the header returns results without full-page reload.

### Changes

**`courses/views.py`** — new view:
```python
from django.contrib.postgres.search import SearchVector, SearchQuery, SearchRank

@login_required
def recherche_view(request):
    query_str = request.GET.get("q", "").strip()
    results = []
    if len(query_str) >= 2:
        query = SearchQuery(query_str, config="french")
        vector = SearchVector("titre", weight="A", config="french") + \
                 SearchVector("contenu", weight="B", config="french")
        results = (
            Lecon.objects
            .filter(chapitre__niveau=request.user.niveau)
            .annotate(rank=SearchRank(vector, query))
            .filter(rank__gte=0.01)
            .select_related("chapitre__matiere")
            .order_by("-rank")[:20]
        )
    return render(request, "courses/recherche.html", {"results": results, "query": query_str})
```

## Task 31 — PDF Export of Lessons

**Problem:** Students study offline (no connection at school/exam). No downloadable version of lessons.

**Approach:** `weasyprint` renders the lesson's HTML content to PDF on the fly. No static files generated — PDF is streamed as a response.

@login_required
def lecon_pdf_view(request, lecon_pk):
    lecon = get_object_or_404(Lecon, pk=lecon_pk)
    if not request.user.is_admin and lecon.chapitre.niveau != request.user.niveau:
        return redirect("matieres")

    from weasyprint import HTML
    from django.http import HttpResponse
    from django.template.loader import render_to_string

    html_string = render_to_string("courses/lecon_pdf.html", {
        "lecon": lecon,
        "contenu_html": lecon.contenu_html,  # already rendered Markdown
    }, request=request)

    pdf_bytes = HTML(string=html_string, base_url=request.build_absolute_uri("/")).write_pdf()
    response = HttpResponse(pdf_bytes, content_type="application/pdf")
    response["Content-Disposition"] = f'attachment; filename="{lecon.slug}.pdf"'
    return response

## Task 34 — Database Backups

**Problem:** No scheduled PostgreSQL backups on Heroku. A bad migration or accidental data deletion is unrecoverable.

**Approach:** Heroku Postgres `pg:backups:schedule` (built-in, no extra cost on Standard+ plans). Also document manual backup/restore procedure.

### Changes — No code changes

**Heroku CLI commands (run once from local):**
```bash
heroku pg:backups:schedule DATABASE_URL --at "02:00 Europe/Paris" --app <app-name>
heroku pg:backups:info --app <app-name>
```

**Document restore procedure in `README.md` or `runbook.md`:**
```bash
# Download latest backup
heroku pg:backups:download --app <app-name>

# Restore to local
pg_restore --verbose --clean --no-acl --no-owner -h localhost -U postgres -d sciencelycee latest.dump
```

**Note:** Heroku's free `postgres:essential-0` includes 1 manual backup. For scheduled daily backups, `postgres:essential-1` (~$9/mo) is required or use a free alternative like `pg_dump` via a Heroku Scheduler dyno + AWS S3.

### Tests to add
None — infrastructure only.
